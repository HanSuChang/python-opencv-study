# OpenCV 실습 프로젝트 모음

OpenCV의 핵심 기능을 활용한 3가지 실습 과제 모음입니다.

---

## 실습 01. 그림판 만들기

마우스 콜백(Mouse Callback) 기능을 활용하여 사용자가 화면 위에 직접 선을 그릴 수 있는 간단한 그림판 구현 실습입니다.

### 주요 기능

- 마우스 이벤트 처리: `cv2.setMouseCallback`을 통해 클릭, 이동, 떼기 이벤트를 감지합니다.
- 연속 선 그리기: 마우스를 누른 채 움직일 때 이전 좌표와 현재 좌표를 `cv2.line`으로 연결하여 끊김 없는 선을 그립니다.
- 캔버스 초기화: `C` 키를 누르면 넘파이 슬라이싱(`img[:] = 255`)으로 화면을 즉시 초기화합니다.
- 실시간 렌더링: `cv2.waitKey(1)`을 사용하여 키 입력과 마우스 동작을 실시간으로 반영합니다.

### 사용된 핵심 API

| API | 설명 |
|-----|------|
| `cv2.setMouseCallback()` | 마우스 동작 감지 및 핸들러 연결 |
| `cv2.line()` | 두 점 사이의 선 그리기 (LINE_AA 안티앨리어싱 적용) |
| `np.ones()` | 흰색 배경 캔버스 생성 |

### 조작 방법

| 입력 | 동작 |
|------|------|
| 마우스 왼쪽 드래그 | 선 그리기 |
| `C` | 캔버스 초기화 |
| `Q` / `ESC` | 프로그램 종료 |

## 실습 02. 크로마키 합성

그린 스크린(Green Screen) 배경에서 촬영된 인물 사진을 추출하여 새로운 배경 이미지에 합성하는 실습입니다.

### 처리 워크플로우

1. 이미지 전처리: 배경 이미지 크기에 맞춰 인물 이미지를 리사이즈합니다.
2. 색상 공간 변환 (BGR → HSV): 빛의 영향을 덜 받는 HSV 색상 공간으로 변환하여 초록색 배경 범위를 정확하게 지정합니다.
3. 마스크 생성: `cv2.inRange()`로 초록색 배경을 추출하고 반전시켜 인물만 선택하는 마스크를 생성합니다.
4. 비트 연산: `bitwise_and`로 인물 이미지에서 사람만 떼어내고, 배경 이미지에서 인물이 들어갈 자리를 검게 처리합니다.
5. 이미지 합성: `cv2.addWeighted()`로 두 이미지를 결합하여 최종 결과물을 생성합니다.

### 사용된 핵심 API

| API | 설명 |
|-----|------|
| `cv2.cvtColor()` | 색상 공간 변환 |
| `cv2.inRange()` | 특정 색상 범위의 마스크 생성 |
| `cv2.bitwise_and()` | 마스크를 이용한 이미지 추출 및 합성 준비 |
| `cv2.addWeighted()` | 두 이미지의 픽셀 값 결합 |

### 실행 결과

| 결과 창 | 설명 |
|---------|------|
| `1_Extracted Person` | 초록 배경이 제거된 인물 이미지 |
| `2_Lena with Hole` | 인물이 들어갈 자리가 비어있는 배경 이미지 |
| `3_Final Result` | 합성 완료된 최종 크로마키 이미지 |

## 실습 03. 문서 스캐너

기울어진 각도에서 촬영된 문서의 네 모서리를 마우스로 선택하여 정면에서 바라본 듯한 반듯한 이미지로 보정하는 문서 스캐너 구현 실습입니다.

### 주요 기능

- 화면 맞춤형 리사이징: 고해상도 이미지가 모니터 밖으로 나가지 않도록 가로 길이 기준으로 비율을 유지하며 리사이즈합니다.
- 마우스 이벤트 좌표 수집: 문서의 네 꼭짓점을 클릭하면 좌표를 저장하고, 클릭 지점에 빨간색 원으로 시각적 피드백을 제공합니다.
- 원근 변환: 사용자가 클릭한 4개의 좌표를 기반으로 변환 행렬을 계산하여 이미지를 반듯하게 펴줍니다.

### 사용된 핵심 API

| API | 설명 |
|-----|------|
| `cv2.getPerspectiveTransform()` | 4개의 점 쌍으로부터 원근 변환 행렬 추출 |
| `cv2.warpPerspective()` | 이미지에 원근 변환 적용 |
| `cv2.setMouseCallback()` | 마우스 클릭 좌표 수집 핸들러 설정 |

### 설정값

- 결과 이미지 크기: 600 x 800 픽셀

### 클릭 순서 (반드시 준수)

정교한 스캔 결과를 위해 아래 순서대로 클릭해야 합니다.

| 순서 | 위치 |
|------|------|
| 1 | 좌측 상단 (Top-Left) |
| 2 | 우측 상단 (Top-Right) |
| 3 | 좌측 하단 (Bottom-Left) |
| 4 | 우측 하단 (Bottom-Right) |


## 실습 04. 레일 중앙선 검출

Canny Edge Detection과 허프 변환(Hough Transform)을 활용하여 레일 이미지에서
좌우 선로를 검출하고 중앙 좌표를 계산하는 실습입니다.

---

### 처리 파이프라인

1. **전처리**: 이미지를 그레이스케일로 변환 후 가우시안 블러로 노이즈를 제거하고,
   Canny 엣지 검출로 윤곽선을 추출합니다.
2. **ROI 설정**: 삼각형 마스크를 적용하여 레일이 있는 하단 영역만 분석 대상으로
   한정합니다.
3. **직선 검출**: `HoughLinesP`로 ROI 내의 직선 후보군을 검출합니다.
4. **선 필터링**: 기울기(slope) 조건으로 수평선 및 노이즈성 선을 제거하고,
   기울기 방향에 따라 왼쪽/오른쪽 레일 선으로 분류합니다.
5. **중앙 좌표 계산**: 좌우 선들의 평균 좌표를 산술 평균하여 레일 중앙선과
   중앙 좌표점을 계산합니다.

---

### 사용된 핵심 API

| API | 설명 |
|-----|------|
| `cv2.Canny()` | 엣지 검출 |
| `cv2.GaussianBlur()` | 노이즈 제거용 블러 처리 |
| `cv2.fillPoly()` | 삼각형 ROI 마스크 생성 |
| `cv2.HoughLinesP()` | 확률적 허프 변환으로 직선 검출 |
| `cv2.line()` | 검출된 레일 선 및 중앙선 시각화 |
| `cv2.circle()` | 중앙 좌표점 시각화 |

---

### 주요 파라미터

| 항목 | 값 | 설명 |
|------|----|------|
| Canny threshold | 60 / 180 | 엣지 검출 민감도 |
| HoughLinesP threshold | 30 | 직선으로 인정하는 최소 교점 수 |
| minLineLength | 60 | 검출할 최소 선 길이 (px) |
| maxLineGap | 150 | 같은 선으로 연결할 최대 간격 (px) |
| slope 범위 | 0.5 ~ 10.0 | 유효한 레일 선의 기울기 범위 |
